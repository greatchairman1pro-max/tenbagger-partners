# 텐베거 파트너스 — 슈퍼디렉터 시스템 워크플로우
## 4인 페르소나 협업 사이클 전체 구조

---

## 페르소나 한눈에 보기

| 페르소나 | 이름 | 역할 | 위치 |
|----------|------|------|------|
| 텐베거업무총괄 | **탑 (TOP)** | 전체 지휘 / 전략 결정 / 추가 수집 지시 | 본사 |
| 데이터수집 전문가 | **킵 (KEEP)** | 갭 분석 / 완성도 점수 산출 / 데이터 정리 | 시스템 |
| 현장디렉터 | **무브 (MOVE)** | 현장 데이터 수집 / 시스템 입력 / 고객 브리핑 | 현장 |
| 개발자 | **크리스 (CHRIS)** | 포트폴리오 생성 / Vercel 배포 / 링크 전달 | 개발 |

## 전체 워크플로우 다이어그램

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
                  1차 사이클
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[무브] 현장 방문
  → 상호명 + 기본정보 + 재무 필수 데이터 수집
  → 시스템에 입력
        ↓
[킵] 데이터 수신
  → 4대 영역 기준 갭 분석
  → 완성도 점수 산출 (0~100점)
  → 갭 분석 보고서 작성
  → 탑에게 보고 / 크리스에게 전달
        ↓
[탑] 보고서 검토
  → 완성도 45점 미만이면 → 무브에게 재수집 지시 (생성 보류)
  → 완성도 45점 이상이면 → 크리스에게 1차 생성 지시
        ↓
[크리스] 1차 포트폴리오 생성
  → 파일명: {상호명}_v1.html
  → 미수집 항목 "데이터 수집 중" 처리
  → Vercel 배포
  → 무브에게 링크 전달
  → 탑에게 1차 완성 보고
        ↓
[탑] 1차 포트폴리오 검토
  → 추가 수집 필요 항목 선별
  → 무브에게 "필수 / 권장" 등급으로 추가 수집 지시

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
                  2차 사이클
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[무브] 탑의 추가 수집 지시 수신
  → 2차 방문에서 "필수" 항목 우선 수집
  → 추가 데이터 시스템 재입력
        ↓
[킵] 추가 데이터 수신
  → 갭 분석 업데이트
  → 완성도 점수 재산출
  → 업데이트 보고서 작성 → 탑 + 크리스에게 전달
        ↓
[탑] 업데이트 보고서 검토
  → 완성도 75점 이상 확인 → 크리스에게 최종 배포 승인
  → 완성도 75점 미만이면 → 무브에게 3차 수집 지시 (사이클 반복)
        ↓
[크리스] 최종 포트폴리오 생성
  → 파일명: {상호명}.html (최종본)
  → 전 섹션 데이터 완비 상태로 생성
  → Vercel 최종 배포
  → 무브에게 최종 링크 전달
  → index.html 사업자 목록에 추가
        ↓
[무브] 최종 포트폴리오 수신
  → 사장님(고객)에게 데이터 기반 브리핑 실행
```

## 완성도 점수 기준 흐름

```
무브 입력 데이터
      ↓
킵이 점수 산출
      ↓
  ┌───────────────────────────────────┐
  │  45점 미만   → 생성 보류           │
  │              → 탑이 무브에게       │
  │                재수집 지시         │
  ├───────────────────────────────────┤
  │  45~74점    → 1차 포트폴리오 생성  │
  │              → 크리스가 _v1 배포   │
  │              → 탑이 추가 지시      │
  ├───────────────────────────────────┤
  │  75점 이상  → 최종 포트폴리오 생성 │
  │              → 크리스가 최종 배포  │
  │              → 무브에게 전달       │
  └───────────────────────────────────┘
```

## Vercel 파일 관리 구조

```
tenbagger-portfolio/ (Vercel 프로젝트)
├── index.html              ← 전체 사업자 목록 + 완성도 현황
│
└── clients/
    ├── 스시오마카세_v1.html  ← 1차 (완성도 58%)
    ├── 스시오마카세.html     ← 최종 (완성도 82%)
    ├── 강남카페모카_v1.html
    └── 강남카페모카.html
```

**파일명 생성 규칙:**
- 무브가 입력한 **상호명** 그대로 파일명으로 사용
- 공백은 언더바(`_`)로 치환, 특수문자 제거
- 1차: `{상호명}_v1.html` / 최종: `{상호명}.html`

## 사이클별 각 페르소나 역할 요약

### 1차 사이클

| 단계 | 페르소나 | 주요 행동 |
|------|----------|-----------|
| 1 | 무브 | 상호명 + 기본 + 재무 데이터 입력 |
| 2 | 킵 | 갭 분석 + 완성도 점수 산출 |
| 3 | 탑 | 전략 검토 + 1차 생성 지시 or 보류 |
| 4 | 크리스 | 1차 포트폴리오 생성 + Vercel 배포 |
| 5 | 탑 | 1차 검토 + 무브에게 추가 수집 지시 |

### 2차 사이클

| 단계 | 페르소나 | 주요 행동 |
|------|----------|-----------|
| 1 | 무브 | 추가 데이터 수집 + 재입력 |
| 2 | 킵 | 갭 분석 업데이트 + 점수 재산출 |
| 3 | 탑 | 75점 이상 확인 + 최종 배포 승인 |
| 4 | 크리스 | 최종 포트폴리오 생성 + Vercel 배포 |
| 5 | 무브 | 사장님 브리핑 실행 |
